Image page.

A specimen page for Machine View. It is deliberately image-heavy, and every image stands for a different case. Side by side it becomes visible which images exist for a machine and which do not. The faults on this page are intentional, which is why it is excluded from indexing.

One: image with descriptive alt text

It arrives, as exactly that one sentence. Everything else, composition, colour, effect, does not.

Two: a claim set as an image

The most common case in practice. The most important sentence on the page sits inside the image, not in the text. The alt text reads "Campaign visual". Formally the image is labelled, and the claim itself still arrives nowhere.

Three: an image series with no alt text

Three images with no alt attribute. In the tree they appear as nameless nodes. A system knows something is there, but not what.

Four: decoration, correctly hidden

An empty alt plus role="presentation". This is not a fault but the correct treatment for pure decoration. The image disappears on purpose.

Five: background image via CSS

This paragraph sits on an image that exists only in the stylesheet.

An image that does not appear in the document at all. There is no node, no name, no way to label it. For any machine, there is nothing here.

Six: image as a link, with no name

The worst case. The image is operable but unnamed. In the DOM view it shows as a link with no label, and an agent can only guess where it leads.

Seven: icon as SVG, with no title

Icon with no title, deliberately marked as hidden here.

An SVG with no title is empty to a machine. With aria-hidden that is honest; without it, it is a silent loss.

What the comparison shows

Of the seven image cases on this page, one arrives carrying its meaning. One disappears deliberately and correctly. Five are lost.

This is not a badly built page, it is an ordinary one. These cases are everyday practice: claims inside images, galleries without labels, backgrounds in the stylesheet, linked images without names. Anyone running a brand visually runs a large part of it outside what machines can reach.

The conclusion is not to describe every image at length. Alt text is meant to be functional, not exhaustive. The conclusion is that load-bearing statements must not live in the image alone. Whatever is meant to hold has to exist as text as well.

To check: open this page in Machine View and pick "Image page" in the page selector.
